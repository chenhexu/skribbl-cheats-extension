---

Skribbl Auto Guesser is a helper for skribbl.io. It reads the game hint from the page, matches it against a built-in word list, and lets you send guesses by clicking suggestions or by running an automatic guesser with a delay you control. In addition, the Auto-Drawer feature can process an image and draw it on the game canvas for you.

HINT PARSING AND WORD LIST
• Reads the current hint (e.g. "c_t", "_____", or length-only) from the skribbl.io UI and filters a large built-in word list so only matching words are shown.
• Supports underscore patterns, letter counts, and multi-word patterns (e.g. "5-8" for hyphenated words). Words are ranked and shown in a scrollable grid.
• You can add your own words, remove words, and deduplicate the list. The word list is stored locally in your browser.

MANUAL GUESSING
• Click any suggested word to send it as your guess in the chat.
• Use the search box to filter candidates by typing; the grid updates as you type.
• Press Enter to send the first matching result, or press 1–9 to send the 1st–9th result without clicking.

AUTOMATIC GUESSING (BOT)
• **Start** — Runs the bot: it sends guesses from the candidate list one at a time, with a delay between each guess.
• **Stop** — Pauses the bot.
• **Delay (ms)** — Sets the time between automatic guesses (0–8000 ms).
• **Refresh parse** — Re-reads the hint from the page and rebuilds the candidate list and queue.

DOT PREFIX AND CLOSE-HIT
• With "Dot prefix" enabled, guesses are sent with a leading dot (e.g. ".word"). If the game replies "word is close!", the extension can resend the word without the dot so you do not use an extra guess.
• A dedicated close-hit watcher runs right after each dot-prefixed send and can resend without the dot when it sees the "is close!" message. You can set how often it checks (close-hit scan rate, in the details section).
• "Bot refresh rate" (in details) controls how often the extension checks the page for hint and chat changes. "Close-hit scan rate" controls how often the close-hit watcher checks after a dot send.

AUTO-DRAWER (V2)
• **Process image** — Converts your uploaded image into drawing points. Use after adjusting Density and BG removal (see below).
• **Start drawing** — Sends the drawing to the game canvas via the browser’s debugger (CDP) or socket. The bot fills shapes row-by-row and prefers horizontal/vertical strokes for speed.
• **Stop** — Stops the current drawing run.
• **Density** (1–6) — How many sample points per area: 1 = fewer points (faster, less detail), 6 = most detail. Capped to reduce lag.
• **BG removal** (0–120) — Makes near-white pixels transparent so they are not drawn. 0 = off; higher = more aggressive removal. Helps with images on white backgrounds.
• **Calibrate** — Draws a few test dots on the canvas so you can verify the drawing path is aligned (useful after resizing the game window).
• **Where is canvas?** — Logs detected canvas elements on the page; useful if drawing does not appear.
• The Auto-Drawer panel can be dragged out into a separate floating window, resized from any edge or corner, and collapsed to a header bar. Use **Dock back** to return it to the main panel. When docked, it keeps its colors (blue/green/red buttons, etc.).

PROTECTION AND ROUND HANDLING
• The extension detects game phase (guessing, drawing, between rounds) and only runs the bot when you are in the guessing phase. It stops or pauses when you are drawing or when the round ends.
• It detects when you or others guess the word, when the round ends ("the word was"), and when a new round starts. It resets its state so it does not resend old words in the next round.
• If the game shows a spam warning in chat, the extension applies a cooldown and, after repeated detection, stops the bot to reduce the risk of being kicked.

PANEL AND SETTINGS
• The main panel shows the current hint, the search box, and the grid of suggested words. It can be dragged, resized from any edge or corner, and **collapsed** to a title bar using the **−/+** (dim) button in the top-right of the panel.
• **Show dev** (in the header, next to the dim button) — Opens the optional Developer Panel in a separate floating window. The Developer Panel shows a log of bot events, your detected nickname, and tools: Snapshot state, Force hide/show words, Clear sent set, Simulate new round, Clear log, Copy log. The log resizes with the panel. Useful for debugging or checking behavior.
• A "Details" section (expandable via "Show details v" / "Hide details ^") shows status, raw hint, underscore pattern, letter pattern, word count, candidate count, last guess, mode, game phase, close-hit state, and spam cooldown. It also has the word add/remove/dedupe controls and the refresh rate and close-hit scan sliders.

PRIVACY AND SCOPE
• The extension runs only on skribbl.io. It does not send your data to any external server. Settings, word list, and panel positions are stored locally in your browser (localStorage).
• The "debugger" permission is used only for the Auto-Drawer when drawing via CDP on the skribbl.io tab; no data is sent elsewhere.---
